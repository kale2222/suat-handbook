# 设计风格决策与改进规划

> 本文是 [DESIGN-apple.md](./DESIGN-apple.md) 与 [DESIGN-notion.md](./DESIGN-notion.md) 两份参考稿的选型结论与落地计划。
> 决策日期：2026-06-16 ｜ 状态：待实施

## 一、决策：采用 Notion 设计语言（保留深理工紫为主色）

### 结论

为深理工手册采用 **Notion 风格**，并将其唯一强调色由 Notion 蓝替换为既有品牌色**深理工紫 `#75207d`**。

### 为什么不是 Apple

一句话：**Apple 的设计为"展示产品"服务，本项目是"让人读信息、快速查信息"——目标相反。**

| 维度 | 本项目需求 | Apple | Notion |
|---|---|---|---|
| 本质 | 文档/手册，读与查 | 为产品摄影做画框，"博物馆画廊" | "光线好的整洁书桌"，为长文档而生 |
| 信息密度 | 急事快查，要密、扫得快 | 刻意极低，一屏一块 | 留白但信息有组织 |
| 依赖什么 | 功能性截图（地图/快递站/缴费） | 重度依赖高品质产品渲染图，无图则垮 | 截图放圆角框即可，不依赖大图 |
| 阅读体验 | 看操作步骤、弱网手机 | 56px 大标题、整屏滚动 | 暖白纸感画布 + 16px 正文，为读而调 |
| 框架契合 | Docusaurus（文档框架） | 全出血交替色块与文档布局打架 | 文档范式天然吻合 |

**致命点**：Apple 整套系统建立在"虔诚展示产品照片、UI 隐到看不见"之上。本项目没有产品照片，只有功能性信息图，放进 Apple 的画廊画框会很尴尬；且其低密度布局会让一本手册需要无穷滚动。

**Notion 的契合点**："makes long pages feel like a document rather than an app" 正是用户焦虑查信息时想要的感觉——像翻一份可信文档。暖白画布的"平静"对刚被陌生信息淹没的新生也更亲和。

### 从 Apple 唯一保留的纪律

"整个系统只有一个强调色、克制到极致"。此点 Notion 同样具备，故无冲突——两边都强化了这条原则。

## 二、核心原则（改造时不可破的底线）

1. **暖白画布是灵魂**：页面底色用 `#f6f5f4` 暖白，而非纯白；卡片/输入框才用纯白，形成柔和的图底关系。
2. **单一强调色 = 深理工紫**：链接、主按钮、激活/焦点态只用 `#75207d`，不引入第二结构色。
3. **多彩"贴纸"色只装饰、不结构**：分类色点、emoji、插画可用多色，但绝不给 CTA 或结构件上色。
4. **发丝线 + 几乎看不见的阴影**：卡片默认仅 1px 发丝线；需浮起时用多层极淡阴影，绝不用重投影。
5. **重标题 + 安静正文**：标题字重 700、负字距；正文 400、16px、1.5 行高，对比即层级。
6. **中文优先回退**：字体栈必须带中文回退，Inter 不含汉字；负字距别套到汉字上。

## 三、设计令牌规格（Notion → 本项目映射）

> Docusaurus 底层是 Infima，全靠 CSS 变量驱动，多数改动落在 [src/css/custom.css](../src/css/custom.css) 即可。

### 颜色

| 角色 | 值 | Infima 变量 |
|---|---|---|
| 页面画布（暖白纸） | `#f6f5f4` | `--ifm-background-color` |
| 卡片/面板（纯白） | `#ffffff` | `--ifm-card-background-color` |
| 主强调（深理工紫，**保留**） | `#75207d` | `--ifm-color-primary`（已是） |
| 正文墨色（柔黑非纯黑） | `#1a1a1a` | `--ifm-font-color-base` |
| 次级文字 | `#615d59` | `--ifm-color-emphasis-700` |
| 发丝线 | `#e6e6e6` | `--ifm-toc-border-color` / 卡片 border |

分类"贴纸"色（仅用于分类点/插画，对应当前已落地的四大内容目录）：
🎓 新生指南 天蓝 `#62aef0` ｜ 🏃 办事指南 橙 `#dd5b00` ｜ 🍜 校园生活 绿 `#1aae39` ｜ 📚 学业指南 青 `#2a9d99`。
（紫色为品牌主色，分类色刻意避开紫，以免与"强调=可点击"信号混淆。）

### 字体与排版

- 字体栈：`Inter, -apple-system, system-ui, "PingFang SC", "Microsoft YaHei", sans-serif`（`--ifm-font-family-base`）。
- 正文：`16px / 1.5`（`--ifm-font-size-base: 16px`）。
- 标题：字重 700 + 负字距；中文收敛，h1 约 `-0.5px`（Apple/Notion 拉丁文 `-1px` 不照搬到汉字）。

### 形状与高度

- 圆角：卡片 `12px`（`--ifm-card-border-radius`）、按钮 `8px`、CTA 胶囊全圆。
- 阴影：默认仅发丝线；浮起态用多层近透明阴影叠加。

## 四、分步改进规划

> 原则：**先改全局令牌（一次性影响 80% 页面），再调三个关键界面，最后处理暗色与细节**。每步独立可验证、可单独提交。

### 阶段 0：准备（10 分钟）

- [ ] 新建分支 `design/notion-restyle`，避免在 main 上直接动样式。
- [ ] `npm start` 截图首页 + 一篇文档页 + 移动端窗口，作为改造前对照基线。

### 阶段 1：全局令牌（半天，影响面最大、性价比最高）

只动 [src/css/custom.css](../src/css/custom.css)，不碰组件：

- [ ] `:root` 写入第三节的颜色变量（画布暖白、卡片白、墨色、发丝线）。
- [ ] 引入 Inter 字体（`@fontsource/inter` 或 Google Fonts），设 `--ifm-font-family-base` 含中文回退。
- [ ] 设正文 `--ifm-font-size-base: 16px`、行高 1.5；标题字重/负字距。
- [ ] 设 `--ifm-card-border-radius: 12px`。
- **验证**：文档页应已变成"暖白纸 + Inter"观感，正文舒适、链接为紫。截图对比基线。

### 阶段 2：首页（半天）

动 [src/pages/index.tsx](../src/pages/index.tsx) 与其 module.css：

- [ ] 四个分类卡改为 Notion `feature-card`：白底、12px 圆角、发丝线、无重阴影。
- [ ] 每张卡顶部加分类"贴纸"色块/色点（第三节四色），强化信息架构。
- [ ] 现有 🔥🔗 emoji 保留，作为贴纸палитра的延续。
- [ ] 底部访问统计行保持安静 caption 灰（沿用现 `--ifm-color-emphasis-500`）。
- **验证**：首页四卡风格统一、有分类色识别度；移动端单列堆叠正常。

### 阶段 3：导航与页脚（半天）

通常需 swizzle 或补全局 CSS：

- [ ] 导航栏：纯白纤细、底部发丝线，主 CTA（若有）用紫胶囊。
- [ ] 页脚：暖白 `canvas-soft` 色块，链接列安静排布。
- **验证**：全站上下"画框"与内容画布的图底关系成立。

### 阶段 4：暗色模式同步（半天）

- [ ] `[data-theme='dark']` 改为**暖色暗背景**（如 `#1f1d1b` 一类）而非纯黑，延续纸感。
- [ ] 校验紫色主色在暗背景的对比度（现已有 `#c46fcc` 亮紫变体，复用）。
- [ ] 发丝线、卡片面在暗色下的层次重新取值。
- **验证**：切换暗色不刺眼，文字对比度达标（WCAG AA）。

### 阶段 5：收尾与回归（半天）

- [ ] 移动端、弱网（图片懒加载仍生效）回归。
- [ ] `npm run build` 确认无样式构建问题；`npm run serve` 看真实效果。
- [ ] 改造前后截图对比，沉淀到 PRD 设计章节当依据。
- [ ] 合并分支，按 `./scripts/release.sh` 发版。

## 五、扩展性：如何加一个新分类

> 结论先行：**本设计是"分类无关"的，加分类不会让皮肤返工**。核心令牌（画布、墨色、字体、圆角、品牌紫）与分类毫无关系；侧栏由 Docusaurus 自动生成。加分类主要是内容操作，视觉只需顺手两小步。

### 加一个分类的 checklist

**A. 内容层（必做）**
- [ ] 在 `docs/` 下建分类目录（如 `docs/实习就业/`）。
- [ ] 加 `_category_.json`：`label`（带 emoji，如 `💼 实习就业`）+ `position`（控制侧栏顺序）。
- [ ] 把文章放进去，每篇带 `contributors` frontmatter。
- ➡️ 侧栏**自动收录**，这一步零设计工作。

**B. 落地页露出（可选）**
- [ ] 首页分类卡 [src/pages/index.tsx](../src/pages/index.tsx)：在卡片数组加一个对象（标题/描述/`to`），约 4 行。
- [ ] 如需在 `docs/深理工手册/index.md` 的「常用功能」列出文章，手动加相对路径链接（带 `.md`）。

**C. 视觉（很小）**
- [ ] 从下方"预留分类色板"挑一个色，登记到分类→色映射（见下）。

### 分类 → 贴纸色映射

| 分类 | 色 |
|---|---|
| 🎓 新生指南 | 天蓝 `#62aef0` |
| 🏃 办事指南 | 橙 `#dd5b00` |
| 🍜 校园生活 | 绿 `#1aae39` |
| 📚 学业指南 | 青 `#2a9d99` |
| _（预留）_ | 粉 `#ff64c8` · 深棕 `#523410` · 深橙 `#793400` · 深紫 `#391c57` |

> **实现建议**：皮肤落地时把这张表做成**一处集中的映射**（CSS 变量或一个 TS 常量），加分类时只在这一个地方加一行，颜色就贯穿首页卡色点、分类页眉等所有用到的位置——避免色值散落各处。
> **注意**：分类色刻意避开**品牌紫**（紫="可点击"信号，留给链接/CTA）。

### 唯一需留意的排版点

首页分类卡现为 `col col--3`（一行 4 张正好排满）。当卡片数跨越整行边界时调整每行列数即可：

| 卡片数 | 建议 |
|---|---|
| 4 | `col--3`（4/行，当前） |
| 5–6 | `col--4`（3/行）→ 6 张排成 3+3 |
| 7–8 | `col--3`（4/行）→ 8 张排成两行 4 张 |

> 实现皮肤时可让列数按卡片数自适应，从此加分类不必再调栅格。

### 不会被影响的部分

全局令牌、侧栏（自动生成）、文档页、导航、页脚——**加多少分类都不动**。真正的成本在写新分类的内容，不在皮肤。

## 六、已知风险与对策

| 风险 | 对策 |
|---|---|
| Inter 不含汉字，中文掉到默认字体 | 字体栈必带 `PingFang SC`/`Microsoft YaHei` 回退 |
| 拉丁文负字距套到汉字上显挤 | 标题负字距收敛到 `-0.5px` 或仅对拉丁标题生效 |
| 暗色直接纯黑，失去纸感 | 用暖色暗背景，非 `#000` |
| 与 Infima 默认样式打架 | 优先改 CSS 变量，少写覆盖；必要时才 swizzle |
| 字体文件拖慢弱网首屏 | 用 `font-display: swap`，或仅引子集/系统字优先 |

## 七、不做什么（Non-Goals）

- 不引入 Apple 式全出血交替色块布局——与文档框架冲突。
- 不为追求"高级感"上重投影、渐变背景——违背 Notion 平静哲学。
- 不引入第二结构强调色。
- 本轮不再调整信息架构——四大目录（办事/校园生活/学业/新生指南）已于 commit `2bd7ebd`、`9d0148f` 落地，本次只做视觉皮肤。
